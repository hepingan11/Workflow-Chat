# 自然语言编排语义词典 v0.1

这份词典用于把用户输入的自然语言编排命令，稳定映射为项目内部的任务图字段。

目标不是让 LLM “猜”，而是先定义一套项目自己的编排语言。

## 1. 基本原则

- 自然语言只作为输入层，不直接执行。
- 系统必须把输入先转换成结构化任务图。
- 高风险或高歧义词优先走规则层，而不是只依赖 LLM。
- 语义词典的职责是“标准化解释”，不是“智能推理”。

## 2. 角色引用语义

### 2.1 显式角色引用

- `@operator`
- `@programmer`
- `@product_manager`
- `@customer_support`
- `@ceo`

解析规则：

- `@角色` 表示当前句子或从句的执行角色。
- 一个句子中出现多个 `@角色` 时，优先按动作边界切成多个节点。
- 如果出现 `交给@角色B`、`发送给@角色B`、`让@角色B继续`，应生成 `handoff` 节点。

### 2.2 角色语义字段映射

- `role_key`
- `assignee_role_key`
- `participant_role_keys`
- `handoff_to_role_key`

## 3. 工具引用语义

### 3.1 显式工具引用

- `#{Ai最新新闻获取}`
- `#{视频生成}`
- `#{抖音、小红书发布}`

解析规则：

- `#{工具名}` 直接表示工具节点。
- 一个节点只允许绑定一个主工具。
- 多个工具连续出现时，应拆成多个 `tool` 节点，而不是一个节点里包含多个工具。

### 3.2 工具相关字段

- `tool_id`
- `tool_name`
- `input_template`
- `context_reads`
- `context_writes`

## 4. 时间语义

### 4.1 立即执行类

以下词汇默认表示节点创建后立刻执行，并且只执行一次：

- `马上`
- `立刻`
- `立即`
- `现在`
- `等下`
- `待会儿`
- `稍后`

推荐映射：

- 如果没有明确时间，默认当前时间后 10 分钟
- `trigger.type = "scheduled"`
- `trigger.run_at = 当前时间 + 10 分钟`
- `trigger.cron = 目标时间对应的 minute hour * * *`

如果需要区分：

- `5分钟后`
  - 当前时间 + 5 分钟
- `2小时后`
  - 当前时间 + 120 分钟
- `待会儿` / `马上` / `等下` / `稍后`
  - 当前时间 + 10 分钟

### 4.2 固定时刻类

- `每天早上8点`
- `下午2点`
- `今晚9点`
- `明天上午10点`

推荐映射：

- `trigger.type = "scheduled"`
- `trigger.time = "08:00"`
- `trigger.date` 可选
- `trigger.timezone = "Asia/Shanghai"`

### 4.3 周期执行类

- `每天`
- `每周一`
- `每周一到周五`
- `工作日`
- `每月1号`

推荐映射：

- `trigger.type = "recurring"`
- `trigger.rule = "daily" | "weekly" | "monthly" | "weekdays"`

### 4.4 延后执行类

- `确认后下午2点再发布`
- `等我回复后再执行`
- `审批通过后再继续`

推荐映射：

- 节点自身不立即执行
- 通过 `depends_on_step_ids` 和审批分支控制
- 如有明确时间，再叠加 `run_at = "14:00"`

## 5. 审批语义

### 5.1 审批触发词

- `发给我确认`
- `让我确认`
- `我确认后`
- `审核通过后`
- `没问题后`
- `确认没问题再`

推荐映射：

- 生成 `human_approval` 节点
- `approval.required = true`
- `on_approved_step_ids = [...]`
- `on_rejected_step_ids = []` 或终止节点

### 5.3 消息推送语义

以下词汇只表示把结果发给用户，不表示需要用户确认：

- `发给我`
- `发送给我`
- `推送给我`
- `通知我`
- `告诉我`

推荐映射：

- 生成 `message_push` 节点
- 读取上一步输出或共享上下文
- 通过当前配置的通知渠道发送，例如 Telegram
- 推送完成后如果没有后续节点，则流程直接结束

区分规则：

- `发给我审批`、`发给我确认`、`我确认后` 映射为 `human_approval`
- 单独的 `发给我`、`告诉我结果` 映射为 `message_push`

### 5.2 审批否决类

- `有问题则不发布`
- `如果我不同意就停止`
- `拒绝则不继续`

推荐映射：

- `on_rejected_step_ids = []`
- 或生成 `noop/cancel` 分支

## 6. 交接语义

### 6.1 Handoff 触发词

- `交给@角色`
- `发给@角色`
- `转给@角色`
- `让@角色继续`
- `然后由@角色处理`

推荐映射：

- 生成 `handoff` 节点
- `handoff_to_role_key = 目标角色`
- 同时传递 `shared_context`

## 7. 条件语义

### 7.1 顺序条件

- `然后`
- `接着`
- `完成后`
- `之后`

推荐映射：

- `next_step_ids`

### 7.2 分支条件

- `如果...则...`
- `没问题后...`
- `有问题则不...`

推荐映射：

- `on_approved_step_ids`
- `on_rejected_step_ids`
- 或条件节点

## 8. 输出与上下文语义

### 8.1 输出表达

- `整理成视频文案`
- `生成日报`
- `输出周报`
- `汇总成建议`

推荐映射：

- 节点输出应写入 `shared_context`
- 通过 `context_writes` 标记

### 8.2 输入承接表达

- `根据上一步结果`
- `基于刚才内容`
- `拿这个结果继续`

推荐映射：

- `context_reads`

## 9. 默认规则

如果用户没有明确说明：

- 默认时区：`Asia/Shanghai`
- 默认审批目标：`user`
- 默认 `handoff_strategy = manual`
- 默认 `mode = single_role`
- 默认“立即”类任务只执行一次

## 10. 不建议交给 LLM 自由解释的词

以下词建议优先规则化：

- `马上`
- `待会儿`
- `等下`
- `每天`
- `每周`
- `工作日`
- `确认后`
- `有问题则不`
- `交给@角色`
- `使用#{工具}`

## 11. 后续扩展方向

后续可以扩展更多语义类：

- 失败重试：`失败后重试三次`
- 汇报频率：`每完成一步汇报一次`
- 风险等级：`高风险操作前必须问我`
- 预算限制：`不要超过100元`
- 平台优先级：`优先发小红书，再发抖音`

## 12. 时间表达维护方式

时间表达不要全部堆进一个大 prompt。推荐按下面顺序维护：

- 高频确定表达写入规则层，例如 `N分钟后`、`N小时后`、`待会儿`、`马上`、`等下`、`每天早上8点`。
- 容易变化的提示词要求写入模板文件，不写死在业务函数里。
- 长尾表达和容易误判的案例写入 Few-shot 示例文件，用示例提升 LLM 稳定性。
- LLM 输出必须经过结构化 schema 校验，校验失败时不进入执行链路。

当前时间解析模块位于：

- `apps/api/app/services/time_parser/`

提示词和示例位于：

- `apps/api/app/services/time_parser/prompts/system.md`
- `apps/api/app/services/time_parser/prompts/user.md`
- `apps/api/app/services/time_parser/prompts/examples.json`
